# Command: /spec (Example)

## Purpose

Write a feature spec before any implementation.

## Input

Feature description from user.

## Expected Output

Spec at `docs/specs/<feature>.md` with acceptance criteria.

## Step-by-Step Execution

1. Read `docs/vision.md`
2. Ask clarifying questions
3. Write spec with status Draft
4. Request approval → Approved

## Constraints

- No code in this command
