## File Placement Rules

| What | Where |
|------|-------|
| UI components | `src/app/components/` |
| API routes / controllers | `src/app/api/` |
| Use cases | `src/domain/use-cases/` |
| Entities / models | `src/domain/entities/` |
| Database adapters | `src/infrastructure/persistence/` |
| External API clients | `src/infrastructure/clients/` |
| Unit tests | `tests/unit/` mirroring source path |
| Integration tests | `tests/integration/` |
| E2E tests | `tests/e2e/` |

## Naming Conventions

See `docs/coding-standards.md`.

## Adding New Modules

1. Confirm placement with `docs/architecture.md`
2. Add ADR if introducing new boundary or pattern
3. Update this file if structure changes

## Growing the AI team

- Run `aide persona add` to configure a new personality (Programmer, PM, PO, Designer, QA, Business Analyst)
- Run `aide audit` to check survey fill status and generated-file drift
- Never edit `.aide/state.json` by hand — it is managed by the CLI

## AI Agent Instructions

- Create directories only when spec requires them
- Do not flatten domain and infrastructure layers
- Keep tests adjacent in path structure under `tests/`
